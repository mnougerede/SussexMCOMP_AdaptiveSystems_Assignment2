# Adaptive Systems Assignment 2 — Homeostatic plasticity in CTRNNs

Replication and extension of Williams (2006), *Homeostatic Adaptive Networks*, Chapter 7.

This project implements a Beer-style ray-sensor agent in pure Python, evolves CTRNN controllers to catch falling circular objects, and compares evolvability across four conditions: no homeostatic plasticity (HP), HP active during a developmental phase before each trial, HP active during each trial, and HP active in both phases. Four analyses are added that Williams did not perform: behavioural trajectory inspection of evolved individuals, per-neuron viable-range diagnostics across evolution, a frozen-HP assimilation test, and a population-level search-dynamics analysis.

## Project structure

- `src/` — Python source code, organised by simulator component
- `scripts/` — Experiment launcher, status viewer, and analysis scripts
- `tests/` — Unit tests
- `notes/` — Design decisions, methods log, todo
- `plan/` — Experiment targets
- `report/` — LaTeX source
- `batches/` — Batch provenance records
- `figs/` — Generated figures and data CSVs (regenerated by analysis scripts)
- `experiments/` — Evolutionary run output: checkpoints, history, manifests (not included in repo)

## Running the replication experiment

**Warning: this takes several hours per condition.** The full replication (4 conditions × 10 runs × 200 generations) requires substantial compute time. Use `--n_workers` to parallelise fitness evaluation within each run.

```bash
uv run python scripts/launch_batch.py \
  --batch <batch_label> \
  --conditions no_hp dev_only behaviour_only both \
  --n_runs 5 --base_seed <seed> --n_workers 6 --n_gens 200 --pop_size 30
```

Condition names: `no_hp`, `dev_only`, `behaviour_only`, `both` (or the enum forms `HP_OFF` etc.).

Check progress at any time:

```bash
uv run python scripts/experiment_status.py
```

The launcher is failure-tolerant: re-running with the same `--batch` label resumes incomplete runs and skips completed ones.

**Files created per run:**

| Path | Contents |
|---|---|
| `experiments/<batch>/<run>/config.json` | Full `RunConfig` snapshot |
| `experiments/<batch>/<run>/checkpoint.npz` | Latest population and fitnesses |
| `experiments/<batch>/<run>/checkpoint.rng.json` | RNG state for exact resumption |
| `experiments/<batch>/<run>/history/gen_NNNN.npz` | Per-generation fitness stats and population fitnesses |
| `experiments/<batch>/<run>/best_per_gen/gen_NNNN.npz` | Best genotype per generation |
| `batches/<label>.json` | Batch provenance record |

## Key implementation notes

**HP weight rule:** the plasticity update rule is $\tau_w \dot{w} = \rho |w|$ with absolute value on the weight. The absolute value bars are dropped when copying equations from PDFs and are not always reproduced faithfully by automated tools, which can produce the incorrect form $\rho w$. The implementation uses the absolute value form; see Williams (2006) §7.4.1.1 and `notes/design_decisions.md`.

**GA:** tournament selection K=3, elitism 1, Gaussian mutation. This differs from Williams's roulette + top-5 GA; see `notes/design_decisions.md`.

**Frozen-HP semantics:** in the assimilation test, "frozen" means `hp.enabled = False` after a settling window, holding weights and biases at their HP-driven values without resetting to genotype. Williams calls this adiabatic elimination.

**Condition strings:** internally `HP_OFF`, `HP_DEV_ONLY`, `HP_BEHAVIOUR_ONLY`, `HP_BOTH`; display labels `No HP`, `Dev only`, `Behaviour only`, `Both`. Mapping in `scripts/analysis/plot_utils.py`.
